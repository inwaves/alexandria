# Weak-to-Strong Bootstrapping Analysis Report

## Executive Summary

Tested 7 different capability gap configurations.
**Result: Bootstrapping wins in ALL 7 cases tested.**
Average bootstrap advantage: 6.4%
Range: 2.4% to 10.8%

## Key Findings

### 1. Bootstrapping Consistently Outperforms Direct Supervision
- In all 7 tested configurations, weak→medium→strong beats weak→strong directly
- Even the worst-case bootstrap advantage is +2.4%
- Best-case advantage reaches +10.8%

### 2. Error Pattern Breaking is Key
- Average error overlap with supervisor: Direct 29.9% vs Bootstrap 21.9%
- Bootstrap reduces error overlap in 7/7 cases
- Average error overlap reduction: 8.0%

### 3. Intermediate Supervisor Quality Matters
- Weak supervisor accuracy: 29.6%
- Medium supervisor accuracy: 42.1%
- Quality improvement from weak→medium step: 12.5%

### 4. Early Warning Signals
- Strongest predictor of bootstrap advantage: **weak_accuracy** (r=-0.789)
- weak_accuracy: r=-0.789
- weak_supervisor_accuracy: r=-0.769
- weak_strong_gap: r=0.705
- direct_error_overlap: r=0.653

## Mechanism Analysis

**Why does bootstrapping work?**

1. **Supervisor Quality Improvement**: The medium model, even when supervised by the weak model, still performs better than the weak model directly

2. **Error Pattern Dilution**: Each supervision step dilutes systematic errors rather than compounding them

3. **Capability Gap Reduction**: Smaller gaps at each step (weak→medium, medium→strong) are easier to bridge than one large gap (weak→strong)

## Limitations and Caveats

- **Synthetic setup**: Results may not generalize to real language models
- **Limited configurations**: Only 7 capability gap configurations tested
- **Simplified error model**: Real systematic errors may be more complex
- **No failure cases observed**: May need wider parameter ranges to find bootstrap failure modes
- **Small sample size**: Each configuration averaged over only 5 trials

## Implications for Superalignment

- **Staged supervision shows promise**: Rather than jumping directly from weak to strong capabilities, intermediate steps help
- **Error pattern breaking**: Bootstrapping appears to reduce harmful imitation of supervisor errors
- **Robustness across gaps**: Works across different capability gap configurations
- **Need for failure case analysis**: Future work should identify when/why bootstrapping might fail
